# Memory 模块后端开发文档

## 概述

Memory 模块是系统的核心数据存储和检索组件，负责管理记忆文件、记忆片段、数据源以及与 Character 和 Workspace 的绑定关系。

## 文档结构

本目录包含以下文档：

### 1. MemoryApi.md
**核心接口文档** - 包含 Memory 模块的所有核心接口

- **数据源管理接口**: 获取、创建、更新、删除数据源
- **记忆文件管理接口**: 获取、同步文件
- **记忆片段管理接口**: 获取、创建、更新、删除片段，批量操作
- **绑定关系管理接口**: 获取、创建、更新、删除绑定
- **角色记忆接口**: 角色记忆的管理、检索和运维
- **工作空间记忆接口**: 工作空间记忆的管理、检索和运维
- **检索接口**: 记忆内容的检索和测试
- **运维接口**: 系统状态、检查和维护

### 2. CharacterMemoryApi.md
**Character 模块接口文档** - Character 模块对 Memory 依赖的接口

- **角色记忆管理**: 获取角色记忆列表、详情和内容
- **角色记忆操作**: 创建、更新、删除角色记忆内容
- **角色记忆检索**: 检索角色记忆内容
- **角色记忆运维**: 系统状态、检查和维护
- **绑定接口**: 角色与记忆文件的绑定和解绑

### 3. WorkspaceMemoryApi.md
**Workspace 模块接口文档** - Workspace 模块对 Memory 依赖的接口

- **工作空间记忆管理**: 获取工作空间记忆列表、详情和内容
- **工作空间记忆操作**: 创建、更新、删除工作空间记忆内容
- **工作空间记忆检索**: 检索工作空间记忆内容
- **工作空间记忆运维**: 系统状态、检查和维护
- **绑定接口**: 工作空间与记忆文件的绑定和解绑

### 4. MemoryDataModel.md
**数据模型文档** - 详细描述所有核心数据实体的结构和关系

- **核心数据实体**: SourceDocument、MemoryFile、MemoryChunk、Binding、MountRule、FusedFrom
- **遗留数据模型**: CharacterMemory、WorkspaceMemory、KnowledgeItem（保持兼容性）
- **数据关联关系**: 实体关系图、绑定关系图、数据流动示意图
- **存储设计**: 数据库表结构、索引设计
- **数据约束和验证**: 唯一性约束、引用完整性、数据格式验证
- **数据同步机制**: 同步状态流转、同步策略
- **数据安全考虑**: 加密、访问控制、备份和恢复
- **性能优化**: 查询优化、存储优化、同步优化

## 技术特点

### 1. 数据模型设计

Memory 模块采用层次化数据模型：
- **数据源层**: SourceDocument 代表外部数据源
- **文件层**: MemoryFile 代表解析后的文件
- **片段层**: MemoryChunk 代表最小可检索单元
- **绑定层**: Binding 代表与 Character/Workspace 的关联关系

### 2. 绑定策略

支持三种绑定策略：
- **完全绑定**: 绑定整个文件的所有内容
- **选择性绑定**: 只绑定文件中的特定片段
- **规则绑定**: 基于规则自动选择需要绑定的内容

### 3. 检索能力

提供强大的记忆检索功能：
- **全文搜索**: 基于关键词的检索
- **语义检索**: 基于向量相似度的检索
- **标签检索**: 基于标签的检索
- **组合检索**: 支持多种条件的组合检索

### 4. 运维功能

提供完善的运维功能：
- **系统状态监控**: 运行时状态、同步状态、健康状态
- **系统检查**: 连接性检查、性能测试
- **数据维护**: 索引重建、缓存清除、数据导出/导入

## 使用说明

### 1. 依赖关系

Memory 模块依赖以下模块：
- **Character 模块**: 提供角色管理功能
- **Workspace 模块**: 提供工作空间管理功能
- **检索服务**: 提供记忆检索功能
- **存储服务**: 提供数据存储功能

### 2. 开发流程

1. 阅读数据模型文档（MemoryDataModel.md），了解核心数据实体
2. 阅读核心接口文档（MemoryApi.md），了解基础接口
3. 根据需要阅读 CharacterMemoryApi.md 或 WorkspaceMemoryApi.md，了解模块依赖接口
4. 实现相应的后端接口

### 3. 代码规范

- 接口遵循 RESTful API 设计规范
- 使用统一的响应格式（code-message-data）
- 错误码统一管理
- 接口文档使用 Markdown 格式
